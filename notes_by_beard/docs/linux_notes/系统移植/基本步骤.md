# 基本步骤

## 1.确定目标机、主机之间的连接方式

1. 串口
   1. 速率太低，但是实用性强，大多数开发板都会有
2. usb
   1. 速度快，但是需要移植驱动
3. tcp/ip
   1. 速度快，也是需要移植驱动
4. jtag
   1. 一些arm处理器会有，很方便，但是价格昂贵

## 2.安装交叉编译器

在x86架构的主机上编译的程序是无法在arm开发板上执行的，所以我们需要使用交叉编译器来帮助我们编译源文件。

两种办法安装交叉编译器

1. 使用芯片厂商提供的已经编译好的工具链，下面是这些工具一般情况下的前缀名（前缀后面还会跟上一些东西，才能组成完整的指令，但是我们一般只使用前缀名）：

   1. 第一个词代表目标体系结构（像是平时使用的gcc，默认就是x86的，只是没写）
   2. 第二个词代表厂商名，如果开源的话就是none，很多情况下都是none
   3. 第三个词linux代表这个工具链是专门针对linux操作系统编译的
   4. 第四个词，gnu代表一个开源规则，eabi指的是嵌入式调用标准接口

   `arm-none-linux-gnueabi-`

   `armm-linux`

   `arm-none-eabi-`，==命令中没有操作系统名称，代表不支持操作系统==

   `arm-elf-`，非常老的，现在很少使用

2. 自己动手编译交叉工具链

   1. 这种方法特别麻烦，一般不会这么做
   2. ==一本书：《The  GNU Toolchain for ARM Target HOWTO》==

3. 如果实在没有交叉编译工具，可以尝试搜索一下这个：==codesourcery==

## 搭建主机-目标机数据传输通道

网络相关服务器的配置，嵌入式中，需要配置的两个分别是TFTP和NFS

1. TFTP
   1. FTP的一个简约版，基于UDP协议传输的
2. NFS
   1. 网络文件系统，目的是linux和linux之间做挂载用的
