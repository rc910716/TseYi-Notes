###### driver/char
字符驱动程序，要想修改哪些文件能够编译进内核，需要修改该文件夹下的Makefile文件中的obj-$()变量的值才行，当括号中的值为y时代表编译进内核，设置为m表示不编译进内核但是编译为模块，如果不设置的话就不编译这个模块。
而上述说的变量在什么地方修改？
	在内核根目录下面有一个“.config”文件，这个文件存储了那些变量的值。
而这个“.config”文件在什么地方生成的呢？
	在内核目录下面，执行100ask_imx6ull_defconfig命令，能够将.config文件替换为一种默认开发板的配置文件（也就是命令中提示的开发板对应的配置文件）。
能不能修改.config文件
	可以使用make menuconfig,能够进入图形化配置页面，进行配置。
